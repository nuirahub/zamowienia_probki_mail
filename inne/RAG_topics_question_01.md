Czysty RAG tekstowy (vector search) czasami gubi się w precyzyjnych obliczeniach dat lub sumowaniu kwot – do tego lepiej użyć podejścia hybrydowego (np. function calling do bazy danych + LLM do syntezy tekstu).



Podczas budowania RAG, deweloperzy muszą wiedzieć, jak kierować pytania (tzw. Router).

SQL (Dane strukturalne): Pytania o "ile", "kiedy", "adres", "cena". Tu LLM powinien wygenerować zapytanie SQL do bazy (Text-to-SQL).

Semantyczne (Dane niestrukturalne): Pytania o "co czuje", "podsumuj", "kontekst". Tu LLM szuka w wektorowej bazie danych (Vector DB) fragmentów notatek.

Mieszane (Hybrid): Najtrudniejsze. Np. "Klienci, którzy kupili produkt X (SQL) i narzekali na cenę (Semantyczne)". System musi najpierw wyciągnąć ID klientów z bazy, a potem przeszukać notatki tylko dla tych ID.
